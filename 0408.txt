------------------------------------
NATURAL KEY, SURROGATE KEY
------------------------------------

Natural Key:
A natural key is a real-world attribute that uniquely identifies a record, like an email or Social Security number. It has business meaning but can change over time.

Surrogate Key:
A surrogate key is a database-generated ID with no real-world meaning, like id INT AUTO_INCREMENT. It is stable, small, and used internally to uniquely identify rows.
------------------------
CREATE TABLE employees (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(50),
    department VARCHAR(50),
    salary INT,
    hire_date DATE
);

Query optimization Technique
--------------------------------------------------------
1. use only necessary columns
2. use where before Groupby and Having
3. Avoid functions on indexed columns
   (better use 'between' instead of 'where')
4. use limit efficiently
   (to understand how data distributed... limit data)
    limit 1000
5. use CTE for readable query breakdown
    (debug your query)
6. use EXPLAIN (to know how it works behind)

--------------------------------------------
TRIGGER
--------------------------------------------
A trigger is a special type of stored procedure in SQL that automatically executes when a specific event occurs on a table.

Event can be INSERT, UPDATE, or DELETE
Trigger runs before or after the event

DML Triggers (Data Manipulation Language): data changes such as INSERT, UPDATE, or DELETE.
DDL Triggers (Data Definition Language): schema changes like CREATE, ALTER, or DROP.
